# Groq LLM Integration Implementation Plan

> **For Gemini:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Add a serverless Groq API provider option for LLM synthesis to achieve sub-2-second response times while keeping database retrieval local.

**Architecture:** We will introduce configuration settings for the LLM provider, Groq API key, and model ID. We will implement `GroqChatModel` inside `pipeline/ollama_llm.py` using a lightweight HTTP request to Groq's OpenAI-compatible chat endpoint, and wire the UI to switch LLM instances dynamically based on the configuration.

**Tech Stack:** Python, Requests, Streamlit, Pytest
